CSS Basic Interview Round (Senior → Mid-Level)

1. Explain the CSS Box Model in detail.

The CSS Box Model consists of content, padding, border, and margin. By default (content-box), the width and height apply only to the content area, and padding/border increase total size.

In real-world layouts like cards or form inputs, forgetting box-sizing: border-box can break alignment because padding increases element size unexpectedly.

2. What is CSS specificity and how is it calculated?

Specificity determines which CSS rule gets applied when multiple rules target the same element. Inline styles have highest priority, followed by IDs, classes, and then element selectors.

In large applications, misunderstanding specificity leads to overuse of !important, which creates maintenance problems.

3. Explain difference between relative and absolute positioning.

Relative positioning keeps the element in normal document flow but allows offset movement. Absolute positioning removes the element from flow and positions it relative to the nearest positioned ancestor.

For example, notification badges on cart icons use absolute positioning inside a relative parent.

4. Why does z-index sometimes not work?

z-index only works on positioned elements and is affected by stacking contexts. If a parent creates a new stacking context (via transform or opacity), child z-index cannot escape it.

5. Explain margin collapsing.

When two vertical margins meet, the larger margin wins instead of adding together. This happens only for vertical margins in block-level elements.

This often causes confusion when spacing sections in landing pages.

6. What is the difference between em and rem?

em units are relative to the parent’s font-size, while rem is relative to the root (html) font-size. Using em inside deeply nested components can cause unexpected scaling issues.

7. What is overflow and when is it used?

Overflow controls what happens when content exceeds container size. Common values are hidden, scroll, and auto.

Chat windows and dropdown menus commonly use overflow:auto.

8. Explain display: none vs visibility: hidden.

display:none removes the element completely from layout flow. visibility:hidden hides the element visually but still occupies space.

9. What is inheritance in CSS?

Some properties like color and font-family inherit automatically from parent elements. Layout properties like margin or padding do not inherit.

10. What is the cascade in CSS?

Cascade determines final styles based on origin, importance, specificity, and source order. Understanding cascade helps debug style conflicts efficiently.

11. What are pseudo-classes?

Pseudo-classes like :hover and :focus define element states. They are widely used for user interaction styling.

12. What are pseudo-elements?

Pseudo-elements like ::before and ::after create virtual elements. They are used for decorative UI elements without extra HTML.

13. What are vh and vw units?

vh and vw are viewport-based units. 100vh represents full viewport height.

On mobile, dynamic address bars may affect vh calculations.

14. What is box-sizing and why use border-box?

box-sizing controls how width and height are calculated. border-box ensures padding and border are included inside declared width.

15. How do you center a div using modern CSS?

Flexbox and Grid are preferred for centering. Flex uses justify-content and align-items. Grid uses place-items:center.
