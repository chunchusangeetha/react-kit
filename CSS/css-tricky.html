CSS Tricky Interview Questions (Senior → Mid-Level)

1. Why is position: sticky not working?

Sticky works only when the element has a defined top, left, right, or bottom value. It also fails if any parent has overflow: hidden, scroll, or auto.

In real-world layouts, sticky headers often fail because a wrapper div has overflow:hidden for layout control.

2. Why is z-index not working even after setting a high value?

z-index only works on positioned elements (relative, absolute, fixed, sticky). Also, if a parent creates a new stacking context using transform or opacity, the child cannot escape it.

This commonly happens when modals appear behind other components.

3. Why is percentage height not working?

Percentage height depends on the parent’s defined height. If the parent height is auto, the percentage value has no reference.

This often breaks full-height layouts inside nested containers.

4. Why are flex items overflowing unexpectedly?

By default, flex items have min-width: auto, which prevents shrinking. Setting min-width: 0 allows proper shrinking.

This commonly affects text content inside flex containers.

5. Why does 100vh cause layout issues on mobile?

Mobile browsers include the address bar in viewport height calculation. When the bar hides or shows, layout jumps.

Modern solution: use 100dvh instead of 100vh.

6. Why is margin-top not working on inline elements?

Inline elements ignore vertical margins. To apply vertical spacing, convert them to inline-block or block.

7. Why does text-overflow: ellipsis not work?

For ellipsis to work, the element must have: overflow: hidden; white-space: nowrap; text-overflow: ellipsis;

This often fails because width is not defined.

8. Why is absolute element positioned incorrectly?

Absolute positioning is relative to the nearest positioned ancestor. If no parent has position: relative, it attaches to the body.

9. Why is fixed position not behaving like fixed?

If a parent has transform, it creates a new containing block. The fixed element behaves like absolute inside that parent.

10. Why is there unexpected extra space below images?

Images are inline elements by default. They align with text baseline, creating small bottom gap.

Solution: set display: block or vertical-align: middle.

11. Why does 100vw create horizontal scroll?

100vw includes scrollbar width. This adds extra width causing horizontal scroll.

Better to use width: 100% for full-width layouts.

12. Why is transition not smooth?

Animating layout properties like width or height triggers reflow. Animating transform or opacity is smoother because it uses GPU.

13. Why are grid items overlapping?

If grid-column or grid-row values overlap explicitly, elements stack. Incorrect grid-template definitions also cause this.

14. Why is overflow not scrolling?

Overflow works only if container has fixed height or width. If height is auto, scroll won't appear.

15. Why are margins not applying between flex children?

Flexbox does not collapse margins like block layout. Spacing issues often require gap property instead.

16. Why is background not covering entire element?

Background images require background-size: cover to fill container. Without it, image may repeat or stretch incorrectly.

17. Why does ::before not appear?

Pseudo-elements require content property. Without content: "", they won’t render.

18. Why is position: sticky working only inside one section?

Sticky works within the scrollable parent container. If parent doesn’t scroll, sticky won’t activate.

19. Why is flex: 1 behaving unexpectedly?

flex: 1 is shorthand for flex: 1 1 0. Developers often misunderstand flex-basis value.

20. Why is CSS not applying at all?

Common reasons: Wrong selector, Specificity conflict, File not linked correctly, Browser cache issue.

As a mid-level dev, you should systematically debug using DevTools.
