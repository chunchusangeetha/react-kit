CSS Deep Technical Round

1. Explain Critical Rendering Path.

Browser builds DOM and CSSOM. They combine to form Render Tree. Layout and Paint phases render pixels.

2. What is layout thrashing?

Repeated DOM reads and writes causing multiple reflows. It significantly reduces performance.

3. What is compositing?

Compositing merges painted layers into final screen output. GPU layers improve performance.

4. What is will-change?

will-change hints browser about upcoming changes. Overuse can cause memory issues.

5. What is content-visibility?

content-visibility:auto skips rendering off-screen content. Improves large list performance.

6. What is CSSOM?

CSSOM is CSS Object Model built from stylesheets. It combines with DOM to create render tree.

7. Why inline critical CSS?

Inlining critical CSS reduces render-blocking resources. Improves First Contentful Paint.

8. What creates new stacking context?

opacity < 1, transform, filter, isolation, position + z-index.

9. What is paint containment?

Limits repaint area. Improves rendering efficiency.

10. What is subgrid?

subgrid allows nested grid items to align with parent grid. Useful in complex layouts.
