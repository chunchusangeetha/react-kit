ADVANCED HTML QUESTIONS

1. What are Semantic HTML elements and why are they important?

Semantic elements clearly describe their meaning to both the browser and developers. Examples include: header, nav, section, article, aside, footer.

Benefits:

  • Improves SEO (search engines understand page structure)
  • Improves accessibility (screen readers interpret better)
  • Better code readability and maintainability

2. How does the browser render an HTML page?

Rendering process:

  1. Browser parses HTML → Creates DOM (Document Object Model)
  2. Browser parses CSS → Creates CSSOM
  3. Combines DOM + CSSOM → Creates Render Tree
  4. Layout (Calculates position and size)
  5. Paint (Draws pixels on screen)

Blocking scripts can delay DOM creation, which affects performance.

3. What is the difference between defer and async in script?

Both are used to load JavaScript without blocking HTML parsing.

  • defer: Downloads in parallel but executes after HTML parsing.
  • async: Downloads in parallel and executes immediately when ready.
<script src="app.js" defer></script>
<script src="app.js" async></script>

4. What is the difference between display:none and visibility:hidden?

  • display:none → Removes element from layout (no space occupied).
  • visibility:hidden → Element hidden but space remains.

5. What are void elements?

Void elements are self-closing elements that do not have closing tags.

  • img
  • br
  • hr
  • input
  • meta

6. What is the importance of the alt attribute?

The alt attribute:

  • Improves accessibility (screen readers read alt text)
  • Improves SEO
  • Displays fallback text if image fails

7. What is the difference between localStorage and sessionStorage?

  • localStorage: Data persists even after browser close.
  • sessionStorage: Data clears when tab is closed.

8. What is the difference between id and data-* attributes?

The id attribute uniquely identifies an element. data-* attributes store custom data inside HTML. 

<div id="user" data-role="admin"></div>

9. What is accessibility (a11y) in HTML?

Accessibility ensures websites are usable by people with disabilities. Best practices:

  • Use semantic tags
  • Add alt attributes
  • Use label with input
  • Proper heading structure

10. What is ARIA in HTML?

ARIA (Accessible Rich Internet Applications) attributes improve accessibility when semantic HTML is not sufficient. 

<button aria-label="Close Menu">X</button>

11. What causes layout shift and how to prevent it?

Layout shift happens when elements change position during loading. Causes:

  • Images without width and height
  • Dynamic content injection

Prevention: Always define width and height for images.

12. What is the difference between inline, internal, and external CSS?

  • Inline: Inside element (not recommended)
  • Internal: Inside style tag
  • External: Separate CSS file (recommended)

13. What is progressive enhancement?

Progressive enhancement means building the basic functionality first, then adding advanced features for modern browsers.

14. How can we improve HTML page performance?

  • Minimize DOM size
  • Use lazy loading
  • Compress images (WebP)
  • Use defer for scripts
  • Avoid unnecessary nested divs

15. What is SEO and how does HTML affect it?

SEO (Search Engine Optimization) improves website ranking. HTML helps SEO by:

  • Using proper heading structure
  • Adding meta description
  • Using semantic tags
  • Optimizing images
