Deep Browser Rendering Internals

1. What happens when you type a URL in the browser and press Enter?

Step 1: DNS Lookup

The browser converts the domain name (example.com) into an IP address using DNS (Domain Name System).

Step 2: TCP Connection

The browser establishes a TCP connection with the server (using 3-way handshake).

Step 3: HTTPS Handshake (if secure)

If the site uses HTTPS, SSL/TLS handshake happens to encrypt communication.

Step 4: HTTP Request

Browser sends an HTTP request to the server.

Step 5: Server Response

Server sends back HTML, CSS, JS, images.

2. How does browser convert HTML into a page?

Step 1: HTML Parsing → DOM Creation

The browser reads HTML line by line and builds a tree structure called the DOM (Document Object Model).

Step 2: CSS Parsing → CSSOM Creation

CSS files are parsed to create CSSOM (CSS Object Model).

Step 3: JavaScript Execution

If a script is encountered:

  • Without defer/async → HTML parsing pauses
  • With defer → Executes after parsing
  • With async → Executes immediately when ready

Step 4: Render Tree Creation

Browser combines DOM + CSSOM to create Render Tree. Only visible elements are included. (display:none elements are excluded.)

Step 5: Layout (Reflow)

Browser calculates exact size and position of each element. This is expensive operation.

Step 6: Paint

Browser converts layout into pixels and paints on screen.

Step 7: Compositing

Layers are combined and GPU may be used for faster rendering (especially animations).

3. What is Reflow and Repaint?

Reflow (Layout):

Occurs when changes affect layout structure. Example: - Changing width - Adding/removing element - Changing font size

Reflow is expensive because it recalculates layout for entire or large part of page.

Repaint:

Occurs when visual styles change but layout doesn't change. Example: - Changing background color - Changing visibility

Repaint is cheaper than reflow.

4. What causes layout thrashing?

Layout thrashing happens when JavaScript repeatedly reads layout properties and writes changes in a loop.

Example: 

element.style.width = "100px";
console.log(element.offsetWidth);
element.style.width = "200px";

Reading offsetWidth forces layout recalculation. Doing this repeatedly hurts performance.

5. How to optimize rendering performance?

  • Minimize DOM size
  • Avoid deep nested elements
  • Use transform and opacity for animations
  • Use will-change carefully
  • Batch DOM reads and writes
  • Use requestAnimationFrame

6. What is Critical Rendering Path?

Critical Rendering Path is the sequence of steps browser goes through to convert HTML, CSS, JS into pixels.

Optimizing CRP improves:

  • First Contentful Paint (FCP)
  • Largest Contentful Paint (LCP)
  • Cumulative Layout Shift (CLS)

7. Why is CSS render-blocking?

Browser must download and parse CSS before rendering because it needs styling information for layout.

If CSS is large or slow, rendering is delayed.

8. Why is JavaScript parser-blocking?

When browser encounters script tag (without async/defer), it pauses HTML parsing to execute JavaScript.

Because JS can modify DOM, browser must wait.

9. What is GPU compositing?

Modern browsers use GPU to render layers separately (for example, animations using transform).

Properties like:

  • transform
  • opacity

trigger compositing without reflow. This improves animation performance.

10. What is Shadow DOM?

Shadow DOM allows encapsulation of HTML and CSS. Used in Web Components.

Styles inside shadow DOM do not affect outside elements.

11. What is ARIA?

ARIA (Accessible Rich Internet Applications) improves accessibility when semantic HTML is not enough. 

<button aria-expanded="false">Menu</button>

12. What is role attribute?

Defines the purpose of an element for screen readers. 

<div role="button">Click</div>

13. What is tabindex?

Controls keyboard navigation order.

14. What is XSS?

Cross-Site Scripting is a security vulnerability where attackers inject malicious scripts.

Prevent using: - Proper escaping - Content Security Policy (CSP)

15. What is Content Security Policy (CSP)?

<meta http-equiv="Content-Security-Policy"
content="default-src 'self'">

Restricts which sources can load scripts.

16. Why use rel="noopener noreferrer"?

Prevents security vulnerability when using target="_blank".

17. What is iframe sandbox?

<iframe src="page.html" sandbox></iframe>

Restricts iframe capabilities for security.

18. What is Preload vs Prefetch?

  • Preload: Load important resource immediately.
  • Prefetch: Load resource for future navigation.
<link rel="preload" href="style.css" as="style">
<link rel="prefetch" href="next-page.html">

19. What is srcset and picture?

Used for responsive images. 

<img src="small.jpg"
     srcset="large.jpg 1024w, medium.jpg 768w"
     sizes="(max-width: 600px) 480px, 800px"
     alt="Example">

20. What is robots meta tag?

<meta name="robots" content="index, follow">

21. What is JSON-LD structured data?

Used to provide structured data for search engines.

22. What is preconnect?

<link rel="preconnect" href="https://fonts.googleapis.com">

Improves performance by establishing early connection.

23. What is DNS-prefetch?

<link rel="dns-prefetch" href="//example.com">

Resolves domain early before actual request.
