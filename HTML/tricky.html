TRICKY HTML INTERVIEW QUESTIONS

1. What happens if you forget to close an HTML tag?

Modern browsers automatically try to fix unclosed tags using their HTML parser. However, this can break layout unexpectedly because the browser may restructure the DOM differently than intended.

Example: 

<p>Hello
<div>World</div>

The browser may automatically close the <p> tag before the <div>. Never rely on browser auto-correction.

2. Can we have multiple <h1> tags on a page?

Technically yes, HTML5 allows multiple h1 tags inside different sections.

But for SEO best practice:

  • Use one main h1 per page
  • Use h2, h3 for sub-sections
Search engines use heading hierarchy to understand content structure.

3. What is the difference between strong and b tags?

Both make text bold visually, but they are different semantically.

  • <strong> → Indicates importance (semantic meaning)
  • <b> → Only makes text bold (no semantic meaning)

For accessibility and SEO, always prefer semantic tags.

4. Why should we always use label with input?

Associating label with input improves:

  • Accessibility (screen readers read label)
  • User experience (clicking label focuses input)
<label for="email">Email</label>
<input type="email" id="email">

5. What is the difference between section and div?

<div> is a generic container with no meaning. <section> is a semantic element representing a thematic group of content.

Use section when content has a clear heading and meaning.

6. Why should images always have width and height?

If width and height are not defined:

  • Layout shift may occur
  • Page performance score decreases

Defining dimensions helps the browser reserve space before image loads.

7. What is the difference between id and name attribute?

  • id: Used for CSS/JavaScript targeting (must be unique).
  • name: Used when submitting form data to server.

Without name attribute, form data will not be sent to backend.

8. Why is using too many divs bad?

Excessive nested divs increase DOM size, which:

  • Slows rendering
  • Reduces performance
  • Makes debugging harder

Use semantic tags instead of unnecessary divs.

9. What happens if you put a block element inside an inline element?

Example: 

<span>
  <div>Hello</div>
</span>

This is invalid HTML. Browsers automatically fix it by breaking the structure. It may cause unexpected layout behavior.

10. What is the difference between hidden attribute and display:none?

  • hidden attribute: HTML-based, hides element.
  • display:none: CSS-based, removes element from layout.

Both remove element visually, but hidden is semantic.

11. Why should meta charset be placed at top of head?

The browser reads character encoding early. If placed late, special characters may render incorrectly. 

<meta charset="UTF-8">

12. What is the difference between disabled and readonly?

  • disabled: Cannot edit and not submitted in form.
  • readonly: Cannot edit but value is submitted.

13. Why is using table for layout bad practice?

Tables are meant for tabular data. Using them for layout:

  • Reduces accessibility
  • Breaks responsive design
  • Hard to maintain

Use CSS Flexbox or Grid instead.

14. What is the difference between inline and inline-block?

  • inline: Cannot set width/height.
  • inline-block: Can set width/height.

15. What happens if two elements have same id?

IDs must be unique. If duplicated:

  • JavaScript may select only first match
  • CSS behavior becomes unpredictable
  • Accessibility issues occur
